# Data Provenance

## Provenance Principles

Every derived row should be traceable to source metadata, source retrieval date, transformation version, and reason codes where applicable. The project should support reconstruction of how each issuer-filing moved from candidate universe to filing text, phrase hit, classification, security mapping, price data, and return window.

No missing filing, missing price, failed extraction, or unavailable return window should be silently dropped.

## Source Manifests

Each external source should have a manifest containing:

- source name
- source type
- provider URL or endpoint
- retrieval date
- provider version or data vintage if available
- license or access notes
- retrieval parameters
- raw file paths if downloaded
- checksums for downloaded files

## Firm Universe Provenance

The implemented firm-universe layer is generated by `scripts/build_firm_universe.py`.

Current implemented source label:

- `SEC_CURRENT_LISTED_WITH_SEC_SIC`

Current implemented source inputs:

- SEC current listed ticker feed: `https://www.sec.gov/files/company_tickers_exchange.json`
- SEC company submissions metadata: `https://data.sec.gov/submissions/CIK##########.json`

The SEC submissions metadata is used only for issuer reference metadata such as SIC, SIC description, current tickers, current exchanges, and former company names. This layer does not download filing documents.

Important limitation:

- The implemented universe is a current-listed/current-constituent sample as of `source_date`.
- It is not a fully point-in-time 2015-2025 listed universe.
- Delisted, merged, bankrupt, formerly listed, and renamed firms may be missing unless they remain represented in the current SEC listed feed.
- Historical ticker changes are not fully available from the current SEC sources. The quality output lists multi-ticker issuers and issuers with former company names as review candidates, not as a complete ticker-change history.

Firm identifiers:

- `firm_id` is a stable project identifier based on CIK, formatted as `CIK##########`.
- Ticker symbols are stored as separate identifiers and should not be used as firm IDs.
- The quality output includes `data/quality/firm_identifier_crosswalk.csv` to preserve CIK and ticker identifiers separately.

Firm-universe outputs:

- `config/firm_universe.csv`
- `data/quality/firm_universe_quality_report.md`
- `data/quality/firm_universe_missing_ciks.csv`
- `data/quality/firm_universe_duplicate_tickers.csv`
- `data/quality/firm_universe_ticker_changes.csv`
- `data/quality/firm_universe_ambiguous_firms.csv`
- `data/quality/firm_identifier_crosswalk.csv`
- `data/manifests/source_manifests/firm_universe_sources.json`

Required upgrade before making historical 2015-2025 universe claims:

- Add a point-in-time security master such as CRSP, Compustat, FactSet, Refinitiv, or another historically aware listed-security source.
- Add delisting, merger, bankruptcy, name-change, and ticker-change histories.
- Rebuild the universe using filing-date or issuer-year eligibility rather than current listed status.

## SEC Filing Provenance

Each filing record should preserve:

- CIK
- accession number
- form type
- filing date
- filing year
- report period
- EDGAR acceptance datetime if available
- SEC filing detail URL
- primary document URL
- amendment flag
- source retrieval metadata

Each filing document should preserve:

- source URL
- document type
- sequence
- description
- raw local path once downloaded
- SHA-256 checksum
- content type
- byte size

## Text Extraction Provenance

Each extraction run should preserve:

- extractor name
- extractor version
- run timestamp
- parameters
- input document ID
- extracted text path
- extracted text checksum
- character count
- warnings
- extraction status
- extraction reason code

Section parsing should preserve:

- section label
- section title
- character offsets
- parser confidence
- section status
- section reason code

## Phrase Provenance

Raw phrase hits should preserve:

- phrase taxonomy name and version
- exact phrase from taxonomy
- exact matched text from filing
- normalized phrase
- phrase family
- matcher name and version
- filing section
- excerpt
- character offsets where available
- filing URL
- accession number through filing link

Interpreted classifications should preserve:

- raw phrase hit ID
- classification run name and version
- method type
- model or reviewer metadata if applicable
- label
- confidence or score
- rationale summary where appropriate
- status
- reason code

## Price And Return Provenance

Security mapping should preserve:

- issuer ID
- security ID
- ticker
- exchange
- identifier history
- link method
- link confidence
- source
- link status
- reason code

Price observations should preserve:

- security ID
- price date
- open, close, adjusted close, or total return index fields as available
- volume if available
- currency
- source ID
- price status
- reason code

Return observations should preserve:

- filing ID
- security ID
- return-window definition
- as-of date
- anchor policy
- target end date
- realized start and end price dates
- start and end prices
- computed return
- price source
- computation timestamp
- return status
- reason code

## Audit Trail Expectations

The following joins should always be possible:

- return observation to filing
- filing to CIK, accession number, filing URL, and filing date
- filing to raw source document
- raw hit to exact phrase, excerpt, section, and filing URL
- interpreted hit to raw hit and classification method
- return observation to price source and security mapping
- every failed or unavailable item to a reason code

## Reproducibility Controls

Recommended controls:

- Store immutable raw downloads separately from processed data.
- Use checksums on raw and extracted text artifacts.
- Version phrase dictionaries and classification codebooks.
- Use append-only logs for pipeline events.
- Save configuration files with sample period, as-of date, source names, and provider settings.
- Maintain row-count reconciliation tables or reports at each pipeline stage.
